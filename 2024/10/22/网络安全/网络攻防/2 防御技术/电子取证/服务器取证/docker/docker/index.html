Docker 核心知识与命令

什么是 Docker？

Docker 是一个开源的应用容器引擎，基于 Go 语言并遵从 Apache2.0 协议开源。它可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux/Windows 机器上，也可以实现虚拟化。 核心理念：“Build, Ship and Run Any App, Anywhere” (一次构建，到处运行)。

Docker 的发展与演进

  • 早期探索 (dotCloud)：Docker 最初是 dotCloud 公司创始团队（Solomon Hykes 等）的一个内部项目，基于 Google 推出的 Go 语言开发。2013 年初开源。

  • 技术演进：

    • LXC 时代：早期 Docker 底层依赖于 Linux 特有的 LXC (Linux Containers) 技术。
    • libcontainer：从 Docker 0.9 开始，Docker 开发了 libcontainer 替代 LXC，实现了对底层特性的直接调用。
    • OCI 标准化：随着容器生态的壮大，Docker 剥离了底层运行时，捐献给了 OCI（Open Container Initiative）基金会，演变为今天的 runc 和 containerd 架构。

  • 容器 vs 虚拟机 (VM)：

    • 传统虚拟机通过 Hypervisor 虚拟化硬件，每个 VM 都有完整的 Guest OS（客户机操作系统），资源占用大，启动慢（分钟级）。
    • Docker 容器直接运行在宿主机的内核上（共享内核），没有独立的硬件虚拟化和完整的 OS，极其轻量，启动极快（毫秒级）。

三、 Docker 核心组件与架构

Docker 采用的是 C/S（客户端/服务器）架构。

  1. Docker Client (客户端)： 你敲击 docker run 等命令的地方。客户端通过 REST API 与守护进程进行通信。
  2. Docker Daemon (守护进程/服务端)： 运行在宿主机上的后台进程（dockerd），负责监听客户端请求，并管理 Docker 对象（构建、运行、分发容器等）。
  3. Docker Registry (镜像仓库)： 用于存储 Docker 镜像的服务。官方公开的公共仓库是 Docker Hub。
  4. Docker Image (镜像)： 一个只读的模板，包含了运行应用程序所需的所有代码、运行库、环境变量和配置文件。它是创建容器的“图纸”。
  5. Docker Container (容器)： 镜像的运行实例。你可以把它看作是一个简易版的 Linux 环境（包括 root 用户权限、进程空间、用户空间和网络空间等）以及运行在其中的应用程序。

四、 Docker 底层管理机制 (核心)

Docker 能够实现隔离和资源限制，完全依赖于 Linux 内核的以下三大核心机制：

  1. Namespaces (命名空间) —— 实现“隔离” Namespaces 为容器提供了一个隔离的运行空间，让容器里的进程感觉自己独占了整个系统。

    • PID Namespace：隔离进程号（容器内的进程 PID 可以从 1 开始）。
    • NET Namespace：隔离网络设备、IP、端口等。
    • IPC Namespace：隔离进程间通信。
    • MNT Namespace：隔离文件系统挂载点。
    • UTS Namespace：隔离主机名和域名。
    • USER Namespace：隔离用户和用户组。

  2. Cgroups (控制组) —— 实现“资源限制” 虽然 Namespaces 实现了隔离，但多个容器如果共享宿主机资源，可能会导致资源耗尽。Cgroups (Control Groups) 负责限制、记录、隔离进程组所使用的物理资源（如 CPU、Memory、磁盘 I/O 等）。

  3. UnionFS (联合文件系统) —— 实现“分层存储” Docker 镜像采用了分层构建的机制。UnionFS 允许将不同目录挂载到同一个虚拟文件系统下。Docker 默认的文件系统驱动通常是 overlay2。

    • CoW (Copy-on-Write) 写时复制：当容器启动时，Docker 会在只读的镜像层之上添加一个可读写的容器层。所有的修改都发生在这个顶层，底层镜像始终保持不变，从而极大地节省了存储空间和分发时间。

五、 Docker 命令大全

1. 系统信息 (Info / Version)

  • docker info：显示 Docker 系统信息，包括镜像和容器数。
  • docker version：显示 Docker 版本信息。

2. 镜像操作 (Images)

镜像仓库与拉取：

  • docker login / logout：登录 / 登出 Docker 镜像仓库。
  • docker search <image>：从 Docker Hub 查找镜像。
  • docker pull <image>:<tag>：从镜像仓库拉取或更新指定镜像。
  • docker push <image>:<tag>：将本地镜像推送到远程仓库。

本地镜像管理：

  • docker images：列出本地主机上的镜像。
  • docker rmi <image>：删除本地一个或多个镜像。
  • docker tag <image> <new_tag>：标记本地镜像，将其归入某一仓库。
  • docker build -t <image_name>:<tag> .：通过 Dockerfile 构建镜像。
  • docker history <image>：查看指定镜像的创建历史。
  • docker save -o <file.tar> <image>：将指定镜像保存成 tar 归档文件。
  • docker load -i <file.tar>：导入使用 docker save 命令导出的镜像。
  • docker import <url/file>：从归档文件中创建镜像。

3. 容器操作 (Containers)

生命周期管理：

  • docker run [OPTIONS] IMAGE [COMMAND]：创建一个新的容器并运行。常用参数：-d (后台运行), -p (端口映射), -v (目录挂载), --name (命名)。
  • docker start / stop / restart <container>：启动 / 停止 / 重启一个或多个容器。
  • docker kill <container>：强制停止容器 (发送 SIGKILL 信号)
  • docker rm <container>：删除一个或多个容器（默认只能删除已停止的）。
  • docker pause / unpause <container>：暂停 / 恢复容器内的所有进程。
  • docker create：创建一个新的容器但不启动它。
  • docker exec -it <container> /bin/bash：在运行的容器中执行命令（最常用于进入容器终端）。

状态查看与交互：

  • docker ps：列出当前正在运行的容器 (-a 包含已停止的)。
  • docker inspect <container>：获取容器/镜像的元数据（返回 JSON 格式）。
  • docker top <container>：查看容器中运行的进程信息。
  • docker attach <container>：连接到正在运行中的容器标准输入/输出。
  • docker events：获取实时事件。
  • docker logs <container>：获取容器的运行日志 (-f 跟踪日志输出)。
  • docker wait <container>：阻塞运行直到容器停止，然后打印出它的退出代码。
  • docker export <container> > <file.tar>：将容器的文件系统导出为 tar 归档文件。
  • docker port <container>：列出指定的容器的端口映射。
  • docker stats <container>：实时显示容器资源使用统计（CPU、内存等）。

容器 Rootfs 操作：

  • docker commit <container> <new_image>：从容器创建一个新的镜像（记录你在容器里做的修改）。
  • docker cp <path> <container>:<path>：用于容器与主机之间的数据拷贝。
  • docker diff <container>：检查容器里文件结构的更改。

4. 网络管理 (Network)

  • docker network ls：列出所有网络。
  • docker network create <network_name>：创建一个新的网络。
  • docker network connect <network> <container>：将容器连接到网络。
  • docker network disconnect <network> <container>：断开容器与网络的连接。
  • docker network rm <network>：删除一个或多个网络。

5. 存储卷管理 (Volumes)

数据卷用于持久化容器数据，使其生命周期独立于容器。

  • docker volume ls：列出所有数据卷。
  • docker volume create <volume_name>：创建一个数据卷。
  • docker volume inspect <volume_name>：查看数据卷详情。
  • docker volume rm <volume_name>：删除未使用的卷。

6. Docker Compose (多容器编排)

用于定义和运行多容器 Docker 应用程序的工具，通过 docker-compose.yml 配置文件管理。

  • docker-compose up -d：构建，（重新）创建，启动并附加到容器（-d 后台运行）。
  • docker-compose down：停止并删除容器、网络、镜像和数据卷。
  • docker-compose ps：列出项目中目前的所有容器。
  • docker-compose build：构建或重新构建服务。
  • docker-compose logs：查看服务容器的输出日志。
  • docker-compose start / stop / restart：启动 / 停止 / 重启已存在的服务容器。
  • docker-compose run <service> <command>：在一个服务上执行一次性命令。
  • docker-compose rm：删除已停止的服务容器。
