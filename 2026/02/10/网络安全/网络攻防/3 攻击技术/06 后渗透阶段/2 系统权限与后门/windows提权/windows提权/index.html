用户与用户组

在windows不同用户有着不同的权限，包括：完全控制、修改、读取与执行、列出文件夹内容、读取、写入。而超级管理员和system用户权限最高

内置用户

  • Administrator：系统管理员，拥有完全控制权
  • guest：来宾账号，提供访问共享资源的网络用户使用，仅拥有基本权限，默认被禁用

    • administrators组
    • users组，新建用户默认所属组
    • guests组，权限最低
    • iss__users组，信息服务使用的内置组

windows安全性

Windows 安全性主要依赖于四个基本元素之间的交互：访问令牌 (Access Token)、安全标识符 (SID)、安全描述符 (Security Descriptor)、访问检查 (Access Check)。

1. 访问令牌 (Access Token)：用户的“身份证”

访问令牌是用户登录时系统创建的，伴随用户的所有进程。它本质上定义了两个关键概念：Permission (权限) 和 Privilege (特权)。

1.1 权限 vs 特权 (关键区别)

虽然两者听起来相似，但在 Windows 内核中机制完全不同：

  • Permission (权限/访问权): 是针对对象的。
    • 例如：是否有权“读取”某个文件、是否有权“写入”注册表。它是通过对比令牌与对象的 ACL 来决定的。
  • Privilege (特权): 是针对系统的。
    • 例如：是否有权“更改系统时间” (SeSystemtimePrivilege)、“关闭系统” (SeShutdownPrivilege) 或“调试程序” (SeDebugPrivilege)。特权直接存储在访问令牌中，使用前通常需要代码显式开启（Enable）。

      1.2 令牌的组成

访问令牌包含了关于安全上下文的所有信息：

  • 用户 SID: 用户的唯一标识。
  • 组 SID: 用户所属组（如 Administrators）的列表。
  • 特权列表 (Privileges): 用户拥有的系统能力。
  • 登录 SID: 标识当前的登录会话。
  • 默认所有者/默认组/默认 DACL: 当该用户创建新对象时，默认赋予该对象的安全属性。
  • 令牌类型: 主令牌 (Primary) 或 模拟令牌 (Impersonation)。

2. 安全标识符 (SID)：身份的“编码”

SID 是二进制格式的数据结构，是 Windows 系统中用于唯一识别用户、组或计算机的“身份证号”。

2.1 SID 结构

结构中的第一个值包含 SID 结构的相关信息，其余值按层级排列。

格式示例: S-1-5-32-544

  • S: 代表这是一个 SID 标识。
  • 1: 修订版本号 (Revision Level)。
  • 5: 标识符授权机构 (Identifier Authority)，5 代表 NT Authority。
  • 32: 子授权机构/域标识符 (Sub-authority)，32 代表 Builtin (内置域)。
  • 544: 相对标识符 (RID)，544 代表 Administrators (管理员组)。

2.2 已知 SID (Well-known SIDs)

这是系统预留的、具有固定含义的 SID，分为几类：

  • RID 500-999: 保留给内置账户和组。
    • S-1-5-32-544: Administrators (管理员组)。
    • S-1-5-32-545: Users (普通用户组)。
    • SID ending in 500: 默认的 Administrator 账户。
    • SID ending in 501: Guest 账户。
  • 服务隔离相关的已知 SID (Windows 2003+ 引入):
    • S-1-5-18 (LocalSystem): 拥有本地最高权限。在网络访问时，它代表计算机本身 (DOMAIN\Computer$)。
    • S-1-5-19 (LocalService): 权限受限的服务账户。访问网络时相当于匿名用户 (Anonymous)。
    • S-1-5-20 (NetworkService): 权限受限的服务账户。但访问网络时使用计算机账户 (身份同 LocalSystem)。

3. 安全描述符 (Security Descriptor)：对象的“锁”

每个安全对象（文件、注册表、进程等）都有一个安全描述符。它是连接“用户”和“资源”的桥梁。

安全描述符包含四个主要部分：

  1. Owner SID (所有者): 拥有该对象的用户。所有者永远有权修改 DACL（即可以把权限赋予自己），即使他当前被拒绝访问。
  2. Group SID (组): 主要用于 POSIX 兼容，现代 Windows 较少使用。
  3. DACL (自主访问控制列表): 最核心部分。
    • 决定了“谁”可以对这个对象“做什么”。
    • 包含一系列 ACE (Access Control Entries)。
    • 如果没有 DACL (Null DACL)，则允许所有人完全控制；如果是空的 DACL (Empty DACL)，则拒绝所有人访问。
  4. SACL (系统访问控制列表):
    • 用于审计 (Auditing)。
    • 记录谁成功或失败地访问了对象（例如：记录谁试图删除这个文件）。

4. 访问检查机制 (Access Check)：开锁的过程

当一个用户进程试图访问一个对象时，内核中的 SRM (Security Reference Monitor，安全引用监视器) 会执行访问检查算法。

SRM 会将 访问令牌中的 SID 与 对象 DACL 中的 ACE 进行逐条匹配。

4.1 匹配规则 (Canonical Order)

DACL 中的 ACE 通常按照 Deny (拒绝) 在前，Allow (允许) 在后 的规范顺序排列。

4.2 检查流程

SRM 按照顺序从上到下扫描 DACL：

  1. 显式拒绝 (Explicit Deny):
    • 如果遇到一条 Deny 条目且 SID 匹配（如“拒绝 User A 读取”），SRM 立即停止扫描并拒绝访问。
    • _结论：Deny 权限具有最高优先级。_
  2. 累积允许 (Cumulative Allow):
    • 如果遇到 Allow 条目且 SID 匹配，SRM 会累积授予该条目中的权限。
    • 扫描继续，直到用户请求的所有权限都被满足。
    • 一旦所有请求的权限都集齐，SRM 停止扫描并允许访问。
  3. 隐式拒绝 (Implicit Deny):
    • 如果扫描完整个 DACL，用户请求的权限仍未全部被满足（或者 DACL 为空），则拒绝访问。
    • _结论：默认情况下，未明确允许即为拒绝。_
